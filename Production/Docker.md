Docker is a tool to test software in different types of environments without having that setup physical. Dependencies, libraries and versions, frameworks, OS features and microservices can all create errors. *Avoid "but it works on my machine" situations*

You install an **image** of a hardware setup and run it on a **[[Container]]** on your PC, instead of having to test things on multiple physical machines. Docker packages an application all its dependencies in a virtual container that can run a Linux server. Each container runs as an isolated process the space, and takes up less space than regular virtual machines.
